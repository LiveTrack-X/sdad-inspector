# SDAD Inspector 0.0.2

This regular patch release fixes three owner-visible presentation and evidence
gaps in 0.0.1 while retaining the same read-only inspection and portable update
boundaries.

> **Unsigned portable release:** these archives are not code-signed or
> notarized and are not installers. Your operating system may warn or block
> them. Verify `SHA256SUMS`, inspect the public source, and use them only if this
> limitation is acceptable in your environment.

## Fixed

- The one-line product banner is now a **README-only** introduction asset. The
  app Overview starts directly with the active packet, objective, TODO, and
  evidence instead of spending working space on promotional artwork.
- Selecting an evidence entry now keeps its provenance metadata and shows the
  bounded body immediately below it: formatted Doctor/snapshot JSON, verbatim
  state YAML, or safely rendered repository Markdown.
- On Windows, normal frozen-app startup now notifies Explorer about the exact executable path
  and performs a bounded icon-association refresh. The verified
  automatic-update helper performs the same refresh after replacing the file
  and before relaunch. Failure of this cosmetic operation never blocks startup,
  update success, or rollback.
- The Windows executable still embeds the source SDAD Inspector icon and now
  reports product/file version 0.0.2. CI checks every embedded icon frame against
  `packaging/sdad-inspector.ico`.

The archive member remains exactly `SDAD-Inspector.exe` on Windows (and
`SDAD-Inspector` on macOS/Linux), so the already immutable 0.0.1 automatic
product update can verify and apply 0.0.2 without a filename migration.

## What is included

- One Windows x64 archive, one macOS arm64 archive, and one Linux x64 archive
  built from the exact `v0.0.2` tag on GitHub-hosted runners.
- Exactly one **single portable executable** in each archive, built with CPython
  3.12; a destination computer does not need Python or an adjacent `_internal`
  directory.
- The authenticated bundled official SDAD Protocol `v3.2.2` runtime, the
  built-in `official-sdad-3` `ProtocolAdapter`, and Inspector Snapshot schema 2.
- The Korean/English read-only UI, official
  `Plan → Route → Implement → Verify → Report` rail, current packet/TODO view,
  bounded evidence-document viewer, owner-gate display, Rule 5 proposal export,
  and static-report support.
- `SHA256SUMS`, GitHub artifact attestations, and the bounded frozen-only
  automatic product update flow.

The tagged workflow builds and launches every platform candidate, gives each
archive to a separate clean hosted runner for another smoke launch, then creates
a draft, attests and uploads all assets, and publishes a regular immutable
release only after every required job succeeds.

## Run it

1. Download `SHA256SUMS` and the archive matching your OS and architecture.
2. Verify the archive hash using the commands in the README.
3. Extract its only executable and launch it. On Windows, launching 0.0.2 once
   also requests the targeted same-path icon refresh.
4. Choose or pass an SDAD project root containing `sdad-state.yaml`.

## Important limitations

- This is a regular GitHub Release, not a signing, installer, notarization, or
  broad stable-support claim.
- Explorer can display stale cached artwork before the first 0.0.2 launch when
  a different executable was manually copied over the same path. Extracting to
  a new folder avoids that pre-launch cache history.
- The updater changes only the Inspector portable executable. It does not
  download an SDAD engine or write/migrate the inspected project.
- Windows needs WebView2; macOS assets are Apple Silicon arm64 and unsigned;
  Linux needs a working graphical desktop and EGL/GL/XCB/Qt WebEngine runtime.

Friendly setup, checksum, screenshot, SDAD attachment, icon-cache
troubleshooting, architecture, source-build, and validation guidance is in the
[README](https://github.com/LiveTrack-X/sdad-inspector#readme).
