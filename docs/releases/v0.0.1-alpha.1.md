# SDAD Inspector 0.0.1 alpha

This is the first public, experimental SDAD Inspector build.

> **Unsigned alpha:** these archives are not code-signed or notarized and are
> not installers. Your operating system may warn or block them. Verify
> `SHA256SUMS`, inspect the public source, and use them only if that limitation
> is acceptable.

## What is included

- One Windows archive, one macOS archive, and one Linux archive built from the
  exact `v0.0.1-alpha.1` tag on GitHub-hosted runners.
- Each archive contains exactly one **single portable executable** built with
  CPython 3.12; the destination computer does not need a separate Python
  installation or adjacent `_internal` directory.
- The Korean/English read-only Inspector UI, bounded Doctor integration,
  authenticated bundled official SDAD Protocol `v3.2.2` runtime, and static
  redacted-report support.
- A `SHA256SUMS` file covering all three platform archives.

Extract the archive and run its only file (`SDAD-Inspector.exe` on Windows or
`SDAD-Inspector` on macOS/Linux). Start it with an SDAD project path or open it
without a path and choose the project folder. CI downloads that packaged
archive into a separate clean hosted-runner job and smoke-launches the extracted
file before publication.

## Compatible SDAD projects

The primary baseline is the official
[SDAD Protocol v3.2.2](https://github.com/LiveTrack-X/spec-driven-ai-development/tree/v3.2.2).
The Inspector expects `sdad-state.yaml` at the selected repository root and
reads the active SPEC, TODO, findings, routed Markdown, and Doctor JSON declared
by that project. Its frozen compatibility corpus covers Doctor `3.2.1` and
`3.2.2`, report schemas 1 and 2, and state schemas 1 and 2; the bundled runtime
used by this alpha is `3.2.2`.

## Important limitations

- Hosted-runner build and smoke evidence applies only to these exact unsigned
  archives. It is not a broad stable operating-system support claim.
- There is no installer, automatic updater, signing, notarization, package
  registry publication, or production-readiness claim.
- Inspector never executes validation commands declared by the inspected
  project and does not edit that project. Rule 5 export writes only an inactive
  proposal to a separate owner-selected path after explicit confirmation.
- A future SDAD engine may require a new compatibility release of Inspector.
- The one-file runtime extracts into the operating system's temporary directory
  while running; Linux `noexec` temporary filesystems are not covered.
- Python is embedded, but platform browser/display components are not: Windows
  requires WebView2 and Linux requires a working desktop with EGL/GL/XCB
  runtime libraries.

Full setup, usage, architecture, security boundaries, source-build steps, and
checksum commands are in the
[README](https://github.com/LiveTrack-X/sdad-inspector#readme).
