# SDAD Inspector 0.0.3

This regular patch release makes the portable Inspector easier to start, read,
localize, and update while preserving the same read-only project boundary.

> **Unsigned portable release:** these archives are not code-signed or
> notarized and are not installers. Your operating system may warn or block
> them. Verify `SHA256SUMS`, inspect the public source, and follow your
> organization's software policy.

## Fixed and improved

- The automatic product update now uses an authenticated, idempotent
  **successful-update acknowledgement**. After the replacement starts, the new
  process removes only the exact retained `.previous` rollback file and
  consumes the one-time result marker. The completion notice has a close button
  and also dismisses itself after eight seconds.
- Inspector opens its GUI before asking for a project. It restores the newest
  valid recent project, or shows the in-app chooser on first launch; the native
  folder picker opens only after the user selects **Browse**.
- English, Korean, Japanese, and Simplified Chinese product UI catalogs are
  complete. First use follows the computer locale, while an explicit language
  choice persists in Inspector-owned per-user app data.
- Light/dark theme and 90–150% UI scale persist across launches and changing
  loopback ports. The default scale is 110%, with `Ctrl/Cmd` + `+`, `-`, and `0`
  shortcuts.
- Narrow and desktop layouts avoid the reported command-bar, repository-tree,
  owner-gate, and status-bar text collisions. The Overview distinguishes the
  current TODO from other remaining work, and evidence entries show bounded
  bodies below their provenance metadata.
- The status bar links **Created by LiveTrack** to the official repository.
  Public documentation now includes English, Korean, Japanese, and Simplified
  Chinese guides, the MIT License, and GitHub Sponsors metadata.
- The Windows native builder now selects `npm.cmd` directly, preventing
  `WinError 193` during CPython 3.12 one-file builds. The executable embeds the
  source icon and reports product/file version 0.0.3.

## What is included

- One Windows x64 archive, one macOS arm64 archive, and one Linux x64 archive
  built from the exact `v0.0.3` tag on GitHub-hosted runners.
- Exactly one **single portable executable** in each archive. The destination
  computer does not need Python or an adjacent `_internal` directory.
- The authenticated official SDAD Protocol `v3.2.2` runtime, built-in
  `official-sdad-3` `ProtocolAdapter`, and Inspector Snapshot schema 2.
- `SHA256SUMS`, GitHub artifact attestations, direct candidate smoke tests, and
  separately downloaded archive smoke tests on all three hosted platforms.

The archive member remains `SDAD-Inspector.exe` on Windows and
`SDAD-Inspector` on macOS/Linux. The updater continues to target the **exact executable path**
and never downloads an SDAD engine or writes/migrates the inspected project.

## Run it

1. Download `SHA256SUMS` and the archive matching your OS and architecture.
2. Verify the archive hash with the command in the README.
3. Extract the archive's only executable and launch it.
4. Reopen the latest project, choose a recent project, paste a root, or use
   **Browse** to select a repository containing `sdad-state.yaml`.

## Important limitations

- This is a regular GitHub Release, not a signing, installer, notarization, or
  broad stable-support claim.
- Windows requires WebView2; macOS assets are unsigned Apple Silicon arm64;
  Linux requires a graphical desktop and its EGL/GL/XCB/Qt WebEngine runtime.
- The updater is an unsigned single-file self-replacement flow. One rollback
  backup cannot guarantee recovery from every filesystem, endpoint-security,
  policy, or power-loss failure.
- PyInstaller one-file startup extracts embedded dependencies into an OS
  temporary directory. Linux `noexec` temporary filesystems remain outside the
  tested boundary.

Friendly download, checksum, startup, localization, UI scaling, updater,
troubleshooting, source-build, and compatibility guidance is in the
[README](https://github.com/LiveTrack-X/sdad-inspector#readme).
